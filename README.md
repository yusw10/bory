# bory

## 프로그램 명세
- 대상 OS: Windows 11
- 구현 언어: Java (유지보수를 위한 선택)
- 프로그램 형태: 윈도우 데스크톱 애플리케이션

## 프로그램의 목적
- 던전앤파이터(넥슨) 캐릭터의 데미지를 빠르게 확인하기 위한 도구를 제공한다.
- 기존에는 던담(https://dundam.xyz) 사이트에서 캐릭터별로 개별 조회해야 하므로 공대 인원(최대 12명) 확인 시 매번 반복 검색이 필요하다.
- 화면을 캡쳐하여 공대원 정보를 추출하고, 던담 데이터를 일괄 조회해 공대장의 확인 부담을 줄인다.

## 기능 목록
- 필수 기능
  - 현재 화면 캡쳐
  - 캡쳐 이미지에서 공대원 목록(명성도, 캐릭터 이름, 직업명) 텍스트 OCR 추출
  - 추출 데이터로 던담 사이트에 각 캐릭터를 크롤링하여 데미지 데이터 수집
  - 크롤링된 데미지 결과를 프로그램 UI에 출력
  - 네트워크/외부 사이트 오류 발생 시 사용자 안내 메시지 출력
- 선택 기능
  - OCR 결과 수동 수정 UI 제공 (오타 보정)
  - 던담 요청 실패 시 재시도/대체 API 설정
  - 크롤링 결과를 CSV/JSON으로 내보내기
  - 공대원 슬롯(최대 12명) 프로필을 저장/불러오기

## 제약조건
- Windows 11 환경에서 동작하는 Java 애플리케이션이어야 한다.
- 던담(https://dundam.xyz) 사이트 구조 변화에 대비해 크롤러 요청/파싱 로직을 모듈화한다.
- 네트워크 장애 시에도 프로그램이 중단되지 않고 사용자에게 오류를 안내해야 한다.
- 개인정보 보호를 위해 캡쳐/추출 데이터는 로컬에서만 처리하며 외부 전송을 최소화한다.

## 유저 흐름(간단한 플로우)
1. 프로그램 실행
2. (던파 실행 중) 공대원 목록 창을 화면에 띄운다.
3. 프로그램에서 **검색** 버튼 클릭 → 화면 캡쳐 및 OCR로 캐릭터 정보 추출
4. 추출 정보 기반으로 던담 사이트 크롤링 수행
5. 크롤링된 각 캐릭터의 데미지를 UI에 표 형식으로 표시
6. 던담 장애 등 네트워크 오류 발생 시 안내 메시지를 표시하고 재시도 옵션 제공

## 입력/출력 예시
- 입력(화면 상태)
  - 던파 내 공대원 목록 창이 전체 인원(예: 12명) 포함된 상태로 화면에 노출
- 내부 처리 중간데이터 예시(OCR 추출 결과)
  ```json
  [
    { "name": "보리사랑", "class": "버서커", "fame": 45678 },
    { "name": "세컨캐릭", "class": "마도학자", "fame": 42310 }
  ]
  ```
- 출력(UI 예시)
  ```text
  공대원 데미지 요약
  - 보리사랑 (버서커, 명성 45678): 12.34조
  - 세컨캐릭 (마도학자, 명성 42310): 10.02조
  ```

## 성공 기준
화면 캡쳐→OCR→던담 크롤링을 통해 공대원(최대 12명) 데미지를 UI에 일괄 표시하고, 오류 시 안내 메시지를 제공한다.
